Идеи для новых шагов проекта:

1. # Application parameters configuration
перенос из application.properties в базу данных.
(задача на архитектуру и реализацию технической функциональности).

2. Docker container generation with application and run.

3. Background job implementation.
Выгрузка агриментов в файлы в формате json.
(разработка background job, конфигурация + время выполнения,
многопоточность - задача на каждый файл и использование
ThreadPoolExecutor).

4. Обработка файлов xml с запросами в background job
(файлы с запросами лежат в директории на диске, джоб
считывает их и генерит файл с ответом в том же формате).

5. Генерация предложений по страховке для клиентов.
Бизнес идея: разослать клиентам предложения по страховке.
В файлах на диске лежит информация по клиентам,
джоб читает файлы и генерит на их основе предложения
по страховке путешествий по заданной конфигурации.
(разработка конфигурации генерации предложений,
обработка файлов в нескольких потоках).

6. Создание UI для админ части приложения.
(авторизация пользователей, более сложный UI,
новые методы в REST интерфейсе, MVC, Single Application,
возможно добавление React/Angular библиотеки).

7. Защита админ REST части (/insurance/travel/api/internal)
basic authorization.

8. Реализация нового вида страхования: недвижимого имущества.

9. SQL скрипты с запросами в базу данных по анализу данных.

10. SQL скрипты с запросами в базу данных на исправление данных.

11. Refactoring. Изучите код вашего проекта, составьте список
маленьких атомарных точечных рефакторингов кода, которые вы
хотите выполнить для улучшения кода. Выполните каждый из рефакторингов
в отдельном коммите.
