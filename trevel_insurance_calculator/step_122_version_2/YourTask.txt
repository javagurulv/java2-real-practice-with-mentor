Разделение приложения на модули прошло трудно.
В результате разделения получились следующие модули:
- core_api
- core_common
- underwriting
- core_app

Модуль core_api - содержит API core модуля (DTO + commands).

Модуль core_common - содержит общие части кода такие как
util, коэффициенты и классификаторы, а так же их репозитории.

Модуль underwriting - содержит код расчёта премии.

Модуль core_app - содержит UI + REST + само приложение.
