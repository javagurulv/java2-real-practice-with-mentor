Одно из решений проблемы создания DEPOSIT транзакций с
указанием свойства expenseCategory может быть таким:

- вынести классы доменной модели в пакет domain;
- сделать создание объектов Transaction (билдер) package level access;
- создать публичный класс Factory для создания объектов Transaction
  из вне пакета domain.

Первый шаг - вынести классы доменной модели в пакет domain.

Реализацию этого шага вы можете найти в программном коде.

Так же по пакетам были распределены классы бизнес логики приложения.

Изучите сделанные в программном коде изменения и выполните их
в своём проекте.
